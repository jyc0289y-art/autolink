## 1. 전략적 배경 및 의의

### 1.1 문제의식

진영 대표는 방사선방호 분야에서 고부가가치 기술 자산(HOBIS, GEANT4 시뮬레이션, ICRP107 활용 등)을 보유하고 있으나, 해당 시장은 규모가 작고 수요자 풀이 폐쇄적이다. **특수 고부가가치 기술을 대중 시장으로 확장할 교두보**가 필요했다.

### 1.2 본 BM이 교두보로 기능하는 이유

| 보유 자산 (특수 영역) | 대중 시장에서의 활용 |
|---|---|
| 모듈러 웹앱 개발 (HOBIS) | 소상공인용 대시보드 제작 |
| Playwright 자동화 | 웹 크롤링·데이터 수집 자동화 |
| AI 파이프라인 구축 (CineLink) | 리뷰 감성 분석·자동 리포트 |
| ICRP107 같은 정량 데이터 모델링 | 정량 KPI 대시보드 설계 |
| Claude Max 활용 능력 | 1인 생산성 격차 (시장 대비 5~10배) |

핵심 통찰: **기술 스택은 동일하되 수요층만 전환**한다. 진영 대표 입장에서 추가 학습 비용은 0에 수렴하고, 시장은 100배 이상 확대된다.

### 1.3 본 BM의 외부 가치

- 크몽 등 대중 플랫폼에서 후기·실적 누적 → 향후 SeouLink 산하 SaaS 제품(HOBIS, TEREV 등) 마케팅 자산으로 재활용
- 일반 시장 고객 응대 경험은 SeouLink 브랜드의 B2C 기획 역량으로 환류

## 2. 검토 과정 (의사결정 트레이서빌리티)

### 2.1 기각된 후보: 삭제 데이터 복구 서비스

| 검토 항목 | 결과 |

| 법적 리스크 | 개인정보보호법, 통신비밀보호법, 불법 콘텐츠 신고 의무 |

→ "추가 투자 0, 시간·물리적 제약 없음"이라는 진영 대표의 조건과 정반대.

| 후보 | 적합도 | 비고 |

### 2.3 최종 선택: 자동화 + 대시보드 풀패키지

선택 근거:
1. 일회성 외주가 아닌 **월 구독 매출**이 깔리는 구조
2. 결과물이 코드·웹 — 물리적 인계 부담 zero
3. 다른 외주자가 못 하는 차별화 (대시보드·AI 분석·알림 통합)
4. 진영 대표의 모든 보유 자산 동시 활용

## 3. BM 코어

### 3.1 핵심 가치 제안 (Value Proposition)

> **"매일 PC 켜놓고 엑셀 노가다하던 일을, 자동으로 돌아가는 봇 + 언제 어디서나 보는 웹 대시보드로 바꿔드립니다."**

### 3.2 타깃 고객

- 1차: 1억~10억 매출대 이커머스 셀러 / 다점포 외식·서비스업 사장님 / 마케팅 대행사
- 2차: 부동산 중개·갭투자자 / 인테리어 등 시공업체 / 1인 콘텐츠 사업자
- 공통: "엑셀 자동화 + 사장님이 손 안 대고 보고 싶어함"이 충족되는 곳

### 3.3 3단 패키지 구성

| 단계 | 구성 | 단가 (1회성) | 월 구독 |
|---|---|---|---|
| **Basic** | 자동화 스크립트 + CSV/엑셀 결과 | 30~80만 | 없음 |
| **Standard** | 자동화 + 정적 웹 대시보드 (PC/모바일) | 100~250만 | 3~10만 |
| **Premium** | 자동화 + 실시간 대시보드 + 카톡/이메일 알림 + 다중 사용자 로그인 | 300~700만 | 10~30만 |

### 3.4 월 구독 모델 (운영비 분리)

| 구독 등급 | 월 구독료 | 포함 |
|---|---|---|
| Lite | 3만 | 호스팅·도메인 비용, 모니터링, 월 1회 동작 점검 |
| Standard | 8만 | + 셀렉터 변경 대응, 월 1회 30분 이내 수정 |
| Pro | 20만 | + 무제한 셀렉터 대응, 월 2시간 이내 기능 추가 |

부업 보호선: **Pro 이상은 수주하지 않음.**

## 5. 단가 산정 체계 (정량 루브릭)

### 5.1 설계 원칙

1. **시급으로 사고하되, 고정가로 판매한다.** 시간당 목표 단가 8~15만, 견적은 예상 시간 × 시급 × 1.5(버퍼).
2. **복잡도 등급으로 미리 자르고, 등급 초과는 견적 협의로 분리.**
3. **유지보수는 무조건 분리 과금.** 무료 유지보수는 영혼을 갈아넣음.

### 5.2 6변수 정량 루브릭 (Quantitative Scoring Rubric)

각 변수 1~3점, 합산 6~18점 범위. 같은 의뢰는 언제 평가해도 같은 등급이 나오도록 설계됨.

#### 변수 ① 데이터 소스 수
- 1점: 1개 (한 사이트 또는 한 엑셀 파일)
- 2점: 2~3개 (여러 소스 통합)
- 3점: 4개 이상 (본격적 통합 작업)

> 작업량은 곱셈으로 증가 (인증·에러·형식 통일).

#### 변수 ② 소스 종류
- 1점: 정형 (엑셀, CSV, 공개 API)
- 2점: 반정형 (로그인 없이 접근 가능한 일반 웹)
- 3점: 비정형 (로그인 필요·동적 렌더링·캡차·앱)

> API 30분 vs 로그인+캡차 며칠. 셀렉터 깨짐 빈도도 폭증.

#### 변수 ③ 트랜스폼 로직
- 1점: 단순 가공 (필터·합계)
- 2점: 룰 기반 분기 10개 이하
- 3점: 복잡 분기·도메인 지식·AI 분석

> 룰 분기 10개 초과 시 인터뷰 시간이 작업 시간을 추월.

#### 변수 ④ 출력 형태
- 1점: 엑셀/CSV
- 2점: 정적 대시보드
- 3점: 실시간 + 알림 + 다중 사용자 로그인

> 엑셀 5줄 vs 실시간+로그인은 서버·DB·인증 시스템.

#### 변수 ⑤ 실행 빈도
- 1점: 수동 1회성
- 2점: 일/주 단위 cron
- 3점: 시간 단위·실시간

> 1점 코드 인계, 2점 GitHub Actions, 3점 서버 상시 가동·실패 복구.

#### 변수 ⑥ 클라이언트 데이터 품질
- 1점: 깨끗·문서화됨
- 2점: 일관성 흔들림
- 3점: 더럽고 예외 많음·룰 설명 못 함

> **숨은 시간 빨대의 본진.** 가장 주의해서 매겨야 하는 변수.

### 5.3 점수 → 등급 → 단가 매핑

| 합산 점수 | 등급 | 예시 작업 | 단가 | 작업시간 한도 |
|---|---|---|---|---|
| 6~7 | **S (단순)** | 엑셀 매크로, 단일 시트 자동화 | 10~30만 | 4시간 |
| 8~10 | **A (표준)** | 단일 사이트 크롤링 + 엑셀 | 40~80만 | 12시간 |
| 11~13 | **B (복합)** | 자동화 + 정적 대시보드 + cron | 100~250만 | 30시간 |
| 14~16 | **C (고급)** | 다중 소스 + 실시간 + 알림 + AI | 300~700만 | 60시간 |
| 17~18 | **D (위험)** | 거절 또는 컨설팅 단가 | 1,000만+ 또는 거절 | — |

**경계선 점수는 항상 상위 등급으로 견적.** 의뢰자는 보통 요구사항을 30% 과소평가함.

### 5.4 단가 산정 공식

```
견적가 = (예상 시간 × 시급 8~15만) × 버퍼 1.5
        + 라이선스/외부 비용
        + 플랫폼 수수료(20%)
```

후기 10건 이상 누적 후 버퍼 1.3까지 축소 가능. 본업 컨디션 보호 위해 1.5 미만으로는 내리지 않는다.

## 8. 견적·운영 프로세스 (5단계)

1. **사전 질문지 PDF 발송** — 데이터 소스 URL, 샘플 출력, 실행 빈도, 알림 여부 등 6개 항목
2. **샘플 파일/페이지 1개 수신** — 등급 판정의 필수 입력
3. **15분 화상 통화** — S/A 생략 가능, B 이상 필수
4. **견적서 PDF + 작업 범위 명세서 송부** — 등급, 시간 추정, 포함/미포함 명시 후 서명
5. **계약금 50% 선입금 후 착수** — 크몽 안전결제 활용

이 5단계는 부업 시간 보호의 방어선. 1·2번 생략은 100% 사고로 이어짐.
